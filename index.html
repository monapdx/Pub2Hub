How It Works

  • Generate a Token in Github

    • Login to Github and generate a fine-grained token.

    Enter a word for the token name.

    If you want the token to expire, set an expiration (or choose no expiration).

    Choose which repo you want to start a blog on (or choose All repositories).

    Click +Add permissions and check the boxes next to Actions, Contents, and Workflows.

    Lastly, change Access for each one from Read-only to Read and write.

    Generate token, copy and paste it somewhere safe.

  • Change One Setting in Github Pages

    • Open your repo and go to Settings.

    In the left hand sidebar, click on Pages.

    Under Build and deployment ---> Source, click the dropdown and choose Github Actions.

  • Enter Your Username, Repo & Token

    • Start writing immediately!
